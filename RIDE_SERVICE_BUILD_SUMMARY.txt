API EXAMPLES
================================================================================

Create Ride:
curl -X POST http://localhost:8081/api/v1/rides \
  -H "Content-Type: application/json" \
  -d '{
    "riderId": "RIDER-123",
    "pickupLatitude": 40.7128,
    "pickupLongitude": -74.0060,
    "pickupAddress": "123 Main St",
    "dropoffLatitude": 40.7580,
    "dropoffLongitude": -73.9855,
    "dropoffAddress": "456 Park Ave",
    "passengerCount": 2
  }'

Assign Driver (called by Matching Engine):
curl -X PUT http://localhost:8081/api/v1/rides/RIDE-abc123/driver \
  -H "Content-Type: application/json" \
  -d '{
    "driverId": "DRIVER-456",
    "estimatedFare": 25.50,
    "estimatedDurationSeconds": 600,
    "driverETA": 120
  }'

Get Ride Details:
curl http://localhost:8081/api/v1/rides/RIDE-abc123

Complete Ride:
curl -X POST "http://localhost:8081/api/v1/rides/RIDE-abc123/complete?actualFare=28.75"

Rate Ride:
curl -X POST http://localhost:8081/api/v1/rides/RIDE-abc123/rating \
  -H "Content-Type: application/json" \
  -d '{
    "raterId": "RIDER-123",
    "raterType": "RIDER",
    "rating": 5,
    "feedback": "Excellent service!"
  }'

================================================================================
DATABASE SCHEMA SUMMARY
================================================================================

rides table:
  - Columns: id, rider_id, driver_id, status, coordinates, addresses
  - Fares: estimated_fare, actual_fare
  - Timestamps: matched_at, accepted_at, arrived_at, started_at, completed_at
  - Ratings: driver_rating, rider_rating, feedback
  - Sharding: shard_id (0-7 for horizontal scaling)
  - Indexes: 8 (rider_id, driver_id, status, created_at, shard_id, compounds)

ride_events table:
  - Immutable audit trail of all state changes
  - Columns: id, ride_id, event_type, previous_status, new_status, event_data
  - Indexes: 5 (ride_id, event_type, created_at, initiator_id)
  - Foreign key: ride_id -> rides.id (ON DELETE CASCADE)
