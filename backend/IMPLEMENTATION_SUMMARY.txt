================================================================================
ENDPOINTS IMPLEMENTED
================================================================================

POST /api/v1/auth/login
- Input: phoneNumber, password, deviceId, deviceType
- Output: accessToken, refreshToken, expiresIn, user info
- Rate Limited: Yes (5 attempts per 15 minutes)
- Authentication Required: No

POST /api/v1/auth/refresh
- Input: refreshToken
- Output: new accessToken, expiresIn
- Rate Limited: No
- Authentication Required: No

POST /api/v1/auth/logout
- Input: refreshToken
- Output: success message
- Rate Limited: No
- Authentication Required: Yes

POST /api/v1/auth/verify-mfa
- Input: userId, otp (6 digits)
- Output: success message, tokens
- Rate Limited: Yes (3 attempts per 5 minutes)
- Authentication Required: No

GET /api/v1/auth/validate
- Input: Authorization header with token
- Output: valid (true/false), user_id
- Rate Limited: No
- Authentication Required: Yes (token required)

SECURITY FEATURES
================================================================================

Password Security:
* BCrypt hashing with strength factor 12
* ~200ms per hash (balances security vs. performance)
* No plaintext passwords in database
* Constant-time verification prevents timing attacks

JWT Security:
* HS256 signing algorithm
* SecretKey with 256+ bits
* JTI claim for token identification
* Audience and issuer validation
* Expiration time enforcement
* No sensitive data in claims

Rate Limiting:
* Login attempts: 5 per 15 minutes (Redis backed)
* Device token limit: 3 per user
* OTP attempts: 3 per 5 minutes
* Automatic unlocking after window expires

Token Management:
* Refresh token revocation on logout
* Bulk token revocation on account compromise
* Automatic cleanup of expired tokens (daily)
* Blacklist for JWT revocation
* Device-based token tracking

Additional:
* Stateless authentication (no sessions)
* CORS with origin whitelist
* No CSRF needed (stateless API)
* Audit logging for compliance
* Phone number masking in logs
* Constant-time string comparison for OTP
